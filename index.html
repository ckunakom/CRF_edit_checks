Clinical Data Management: eCRF & Edit Checks

Background

When I was a Clinical Data Analyst (CDA), my task include coming up with all kinds of data cleaning process to ensure that clinical data entered into the database is clean. One of the strategy is to have Edit Checks in place on the eCRF (electronic case report form) in order to eliminate a good chunk of manual data review/data cleaning tasks from the clinical monitors which ultimately saves time and human resources. Additionally, adding validations like Edit Checks enable data cleaning process at a vert early stage, data entry.

The CDA typically creates the pdf version of the CRF and the specification for the Edit Checks, which gestational handed off to the engineers. The engineers then turn those pdf into an electronic form on the web and programmed the validation procedures according to the CDA's specification.

Purpose & Motivation

Since I never had a chance to do this at work, I wanted to try out the following:

  • Turn the pdf version of the CRF to an eCRF
  • Programm the Edit Checks to demonstrate data cleaning at an early stage of data collection

Tools

I use the publicly available pdf version of the Global COVID-19 Clinical Platform: Rapid CRF from the World Health Organization (WHO) as my starting material to develop the eCRF using HTML/CSS/Bootstrap. I changed some of the layout and added some additional questions to incorporate branching logic as I deemed fit. Programmed Edit Checks is done using javaScript. I only did 2 sections for validations since I wanted to start working on other projcets and closed this out.
